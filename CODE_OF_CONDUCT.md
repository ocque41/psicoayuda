# Code of Conduct

We want Nido to be a respectful and useful project for people building
safety-minded software.

## Expected behavior

- Be kind, direct, and practical.
- Assume good intent, but correct problems clearly.
- Respect different levels of experience.
- Keep discussions focused on the project and user safety.
- Be careful with sensitive topics around tragedy, crisis, and mental health.

## Unacceptable behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information.
- Dismissing safety concerns without review.
- Pushing medical claims, emergency guarantees, or unverified emergency data.

## Reporting

For security issues, use [SECURITY.md](SECURITY.md). For conduct concerns,
open a private channel with the project maintainers if one is available. If no
private channel is available yet, avoid posting sensitive personal details in a
public issue.

Maintainers may remove comments, close issues, or block contributors when
needed to protect the project and its users.
